Summary

Methods to read and write text/csv data. Currently, reads from or writes to a String, so only works for data sets small enough to reasonably fit in memory in the application. Current pattern of use would be e.g. 

content err = httpGet "http://example.com/data.csv"

records err = csvRead content

outputContent err = csvWrite records

response err = httpPost "http://example.com/data.csv" "text/csv" outputContent

csvRead - read from a String containing CSV-encoded data - produce a list of data-row records
csvWrite - write a list of data-row records out to a string in CSV encoded form.

csvRead
   content String 
   trimSpace Bool
   fieldsPerRecord Int
> 
   records [] [] String
   err String
"""
 content should contain one or more lines (rows) of CSV-formatted data.
 Reads the CSV-formatted data and returns a list of data-row records. Each
 data-row record is a list of Strings; one String per field.

 See description of ReadAll method in Go "encoding/csv" package for details of operation
 and data accepted.

 If trimSpace is supplied and true, leading and trailing space is trimmed from each field value.

 If fieldsPerRecord is supplied and negative, no check is made and each row (each record) can
 have a different number of fields.

 If fieldsPerRecord is 0 (the default) each record must have same number of fields
 as the first record (first row)

 If fieldsPerRecord is supplied and is a positive integer, it specifies the required number 
 of fields per record
"""

csvWrite
   records [] [] String 
   useCrLf Bool
> 
   content String 
   err String
"""
 Each record (i.e. data row) is a list of Strings, one String per field.
 Writes the records out in CSV format, into the content String.

 See description of WriteAll method in Go "encoding/csv" package for details of operation
 and data accepted.

 If there is an error, the returned content string will be incomplete and
 the second return value will contain an error message.
 If no error, the second return value will be the empty String ""  

 If useCrLf is supplied and true, each line terminates with \r\n instead of
 the default \n.  \r\n is the Windows-compatible text file line ending,
 whereas the default \n is the linux and macosx compatible text file format.
"""
