Summary

Methods of a simple http client. Enables HTTP GET requests and HTTP POST form-posts. You can supply an https url to the methods as well as http.

httpGet - perform a GET request - return the response body or an error string
httpPost - POST some keys and values as in submitting an html form - return response body or an error

Methods


httpGet url String > responseBody String err String
"""
 Performs an HTTP GET request to the specified http or https url.
 Returns the body of the response (not the response headers or trailers) as a String.
 If there is an error, the responseBody will be the empty String "" and err will
 be an error message. If the request was successful, err will be ""

 Follows up to 10 redirects.
"""


httpPost
   url String
   keysVals {} String > Any
> 
   responseBody String 
   err String
"""
 Post the keys and values (as if they were html form-input names and values) to a 
 form-handling web service.

 Performs an HTTP POST request to the specified http or https url, passing
 the (url-arg-encoded) map of keys and values as the content of the POST request.
 Returns the body of the response (not the response headers or trailers) as a String.
 If there is an error, the responseBody will be the empty String "" and err will
 be an error message. If the request was successful, err will be ""

 The keysVals map should be a map from String to (value or collection of values)
 The values are converted to their default String representation to be used as
 posted form-field values.    

 Example:

    responseBody err = 
       httpPost 
          "http://example.com/form-action"
          {
             "firstName" => "Arnold"
             "lastName"  => "Palmer"
             "friend"    => ["Alice" "Bob" "Charlie"]   
          }String > Any


 IMPORTANT LIMITATION! Does not follow redirects. This is a Go http library problem about
 which there is a bug report.
"""
