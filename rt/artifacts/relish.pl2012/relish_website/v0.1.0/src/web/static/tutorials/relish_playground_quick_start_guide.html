Purpose of this Guide

This guide will get you started with finding, invoking, and trying out the relish Simple Program Trial Environment (also known as the Playground).

This interactive environment allows simple relish programs to be typed in quickly and run. This is suitable for learning relish programming, for testing/experimenting with relish program syntax rules, or for testing/learning the functioning of basic program constructs or library methods.

The playground also includes, selectable from a drop-down menu, most of the simple programs examples in a form where you can run them easily.

The playground is probably the thing you should try first when exploring relish. Of course, to really grok what you are seeing in the playground, you will also need to read the relish by-example reference manual.

Obtaining and Running the relish Playground

The relish software artifact that provides the Playground application is described here: http://shared.relish.pl/relish/artifacts/shared.relish.pl2012/dev_tools/metadata.txt

To auto-download and run the playground, open a terminal window, and cd to your relish directory (or a subdirectory). Then issue the following command: 

relish -web 8080 shared.relish.pl2012/dev_tools playground
You should see a confirmation message that the playground has been started. That terminal window will now be "hung" because the playground web app is running and listening for http connections.

Now, enter the following URL into your web browser's address bar: 

   localhost:8080
You should see a multi-pane web page come up, entitled "relish Simple Program Trial Environment".

Instructions - Operating the Simple Program Trial Environment (Playground)

Instructions for operating the environment (the playground) can be found in the /doc/README.txt file in the software artifact. You can find the README file now under your own relish/shared/relish/replicas directory, or at http://shared.relish.pl, specifically here: README.txt.
