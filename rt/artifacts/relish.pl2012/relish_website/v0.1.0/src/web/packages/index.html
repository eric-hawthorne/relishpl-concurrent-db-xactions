You can use the methods from a standard-library package by importing the package as if it were a package in your own software artifact e.g.: 

import 
   strings
Then, in your source code, refer to the methods from the standard-library package without package-name qualification of the method name. For example, s2 = replace s1 "foo" "bar"

relish Built-in Functions

In addition to the small standard library, there is a suite of built-in relish methods which provide useful general-purpose operations on relish primitive data types such as Numeric, Bool, List, Channel, and String. These functions are implemented as native functions, written in Go. These are thin wrappings of functions from several of Go's standard library packages, or of Go primitive operators and built-in functions, such as arithmetic and logical operators.

The currently available Built-in Functions are listed in the relish Syntax and Semantics By Example reference manual.

Roadmap for the relish standard library

The plan is to lift a subset of Go standard library packages into relish by providing thin adapter wrappers for the Go functions. Wrapper functions generally just cast the Go primitive data type arguments into equivalent data types which implement the RObject interface, which every relish data object implements.

A small number of methods will remain as relish language standard built-in methods, available for use in any relish package without importing a package of methods. More of the "lifted" standard library methods, however, will be placed in separate relish standard library packages. The library packages will then have to be individually imported into any new package in order to use the standard library package method.
