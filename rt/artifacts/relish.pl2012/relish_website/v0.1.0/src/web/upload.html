Upload the source-code for your relish software artifact

Upload a metadata.txt file and/or one or more artifact version zip files for a software artifact

  • If uploading files for an artifact that is new to shared.relish.pl, you must upload a metadata.txt file and at least one artifact-version zip file (zipped source code).
  • If updating an already uploaded artifact, you can upload a newer, revised metadata.txt file, and/or you can upload one or more higher-version artifact-version zip files.
  • To make an artifact-version zip file suitable for publishing on your own host server and/or here,
    use relish -publish yourorigin.net2013/your_artifact 0.9.0 to copy the artifact source code (version 0.9.0) from your artifacts directory to your shared/relish/artifacts directory in your local relish installation directory trees.
    relish -publish will create a file called yourorigin.net2013--your_artifact---0.9.0.zip

  • If you upload your relish software artifact here, it will be published on this site, and become part of a searchable and browsable directory of relish software artifacts. As well, relish programs which import your artifact by name will be able to find, download, and link with your software artifact automatically.
  • If you publish your software artifact here, you, or the rightsholder if not you, retain copyright and all other rights to the software artifact, except the right to keep the artifact a trade secret.
